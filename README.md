<h1>Integration of a Emerson RX3i PLC with a Virtual Industrial Process using OPC UA</h1>

<h2>Description</h2>

This project presents the design and integration of a virtual automated production line with a real industrial controller. The main objective was to create a simulation environment that communicates with a physical PLC and allows testing of industrial automation logic in a controlled virtual environment.

The system was developed using the Factory I/O simulation platform connected to an Emerson RX3i programmable logic controller. Control algorithms were implemented in Proficy Machine Edition, enabling the PLC to operate the virtual production line in real time.

Communication between the simulation environment and the controller was established using industrial communication standards and middleware tools such as OPC and Kepware. This allowed the virtual devices in Factory I/O to exchange signals with the physical PLC.

Additionally, the project includes configuration of an HMI interface used for monitoring and controlling the production process. The implemented solution demonstrates practical integration of simulation software, industrial controllers, and communication protocols used in modern automated production systems.

The project demonstrates practical skills in PLC programming, industrial communication, virtual commissioning, and integration of automation systems.
<h2>Communication Architecture Diagram</h2>
<img src="https://i.imgur.com/NdHWVhM.png" width="900"><br><br>                  

The presented diagram illustrates the communication architecture used to integrate the industrial controller with the simulation environment. The central element of the system is the Emerson RX3i programmable logic controller, which executes the control program responsible for operating the production process.

Communication between the PLC and the engineering environment is established using the SRTP (Service Request Transport Protocol) protocol. Through this protocol, the controller exchanges process variables with the Proficy Machine Edition software. In the implemented system, Proficy Machine Edition acts as an OPC UA server, exposing PLC variables as OPC UA nodes.

The simulation environment Factory I/O operates as an OPC UA client, which connects to the OPC UA server and subscribes to selected variables. This mechanism enables bidirectional data exchange between the virtual production line and the real industrial controller. Input signals generated by the simulated sensors in Factory I/O are transmitted to the PLC, while output signals calculated by the control program are sent back to the simulation to control virtual actuators.

Such architecture allows the creation of a hardware-in-the-loop simulation, where a real industrial controller operates a virtual production process. This approach makes it possible to test and verify control algorithms in a safe and flexible virtual environment before deploying them in a real industrial installation.
